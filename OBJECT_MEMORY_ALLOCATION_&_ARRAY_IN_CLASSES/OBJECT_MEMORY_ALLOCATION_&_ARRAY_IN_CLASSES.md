1. Memory Allocation in C++ Objects When you write a class blueprint, it actually takes up zero memory! The compiler only allocates memory when you actually create an Object (like UserAccount obj1;). Here is a brilliant optimization C++ does under the hood: If you create 100 UserAccount objects, C++ gives each object its very own separate memory space for variables (so every user has their own balance). But, to save RAM, C++ only creates one single copy of the class functions in memory, and all 100 objects share those exact same functions!
2. Arrays Inside Classes Just like you can have an int or a string inside a class, you can also put an entire Array inside a class! This is incredibly useful for storing lists of data specific to one object.
