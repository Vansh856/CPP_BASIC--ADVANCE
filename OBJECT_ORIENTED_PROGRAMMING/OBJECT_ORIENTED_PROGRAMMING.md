The Concept & Notes:
Up until now, we have been writing code using a "Procedural" approach—creating variables and then passing them back and forth into completely separate functions. Object-Oriented Programming (OOP) is a paradigm shift. It is a completely different way to think about code. It revolves around bundling your data (variables) and your behaviors (functions) together into a single unit called an Object.
Here are the core pillars of OOP you will need to know:
1. Classes & Objects:
Class: Think of a class as a blueprint or template. It is a custom data type (very similar to the struct you learned earlier) that defines what properties and actions an entity should have.
Object: An object is a real, usable instance created from that blueprint. If "Car" is the class (the blueprint), then your specific "Red Toyota Corolla" is the object.
2. Encapsulation: This is the practice of hiding the internal, sensitive data of an object and only allowing outside code to interact with it through specific, approved functions. It keeps your data safe from accidental changes!
3. Polymorphism: The ability for functions or objects to behave differently depending on the context. You actually already used a form of this in the last lesson with Function Overloading, where the same function name did different things based on its parameters!
4. Inheritance: The ability to create a new class by absorbing the traits of an existing class (e.g., creating a Dog class that inherits all the properties of a broader Animal class).
